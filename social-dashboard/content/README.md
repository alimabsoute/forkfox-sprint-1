# social-dashboard/content/

JSON files in this folder are auto-generated by `forkfox/pipeline/build_social_calendar.py`. Re-run the script to regenerate. Do not hand-edit (changes will be overwritten on next run).

## Expected file shape

Each file is named `day-XX.json` (zero-padded, e.g. `day-04.json`, `day-15.json`) and follows this shape:

```json
{
  "day": 4,
  "date": "2026-05-15",
  "subtitle": "Two scheduled posts · IG + FB",
  "posts": [
    {
      "type": "Feature Spotlight",
      "suggested_time": "11:30 AM ET",
      "scheduled_time": "2026-05-15T15:30:00Z",
      "title": "Tony Luke's · Philly",
      "image_url": "https://forkfox.ai/_assets/posts/2026-05-15-tonylukes.jpg",
      "ig_caption": "Caption text optimized for Instagram...",
      "fb_caption": "Caption text optimized for Facebook (longer, link-friendly)...",
      "hashtags": ["#ForkFox", "#PhillyFood", "#Cheesesteak", "#EatLocal", "#PhillyEats"],
      "image_prompt": "Cinematic close-up of a Philadelphia cheesesteak...",
      "video_prompt": "Slow pan across a counter at a South Philly hoagie shop...",
      "platforms": ["instagram", "facebook"]
    }
  ]
}
```

## Slot types

The `type` field drives badge color in the UI:
- `Feature Spotlight` — red badge
- `Article Reshare` — blue badge
- `Novel Post` — green badge

Any other value renders as a neutral dark badge.

## Optional fields

All slot fields are optional except `type`. The renderer hides any field that is absent. If `posts` is empty or the JSON file is missing, the day page shows a friendly "not yet populated" message.
